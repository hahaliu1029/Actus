## 报告安全漏洞

**请不要通过公开的 GitHub Issue 报告安全漏洞。**

如果你发现了安全漏洞，请通过以下方式私密报告：

1. **GitHub Security Advisory**（推荐）：前往本仓库的 "Security" 标签页，点击 "Report a vulnerability" 提交私密安全报告。

2. **电子邮件**：发送邮件至项目维护者，邮件标题请包含 `[SECURITY]` 前缀。

### 报告内容

请尽可能提供以下信息：

- 漏洞类型（如 XSS、SQL 注入、权限绕过等）
- 相关源文件的完整路径
- 受影响代码的位置（tag/branch/commit 或直接 URL）
- 复现步骤
- 概念验证（PoC）代码（如有）
- 漏洞影响说明

### 处理流程

1. 我们将在 **48 小时内**确认收到你的报告
2. 我们将在 **7 个工作日内**提供初步评估
3. 我们将与你协调修复和披露时间线
4. 修复发布后，我们将公开致谢（除非你要求匿名）

## 安全最佳实践

使用 Actus 时，请确保：

- 不要在 `config.yaml` 或代码中硬编码 API 密钥和密码
- 使用强随机字符串作为 `JWT_SECRET_KEY`（可用 `openssl rand -hex 32` 生成）
- 为 PostgreSQL 设置强密码
- 在生产环境中启用 HTTPS
- 定期轮换 MinIO/S3 访问密钥
- 不要将 `.env` 文件提交到版本控制
